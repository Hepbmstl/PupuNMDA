Other files and folders:
- App.xaml / App.xaml.cs             : WPF startup and resources (themes, styles)
- NeuronCAD.csproj                   : Project file, includes Python scripts as Content to copy to output
- README.md                          : Project README
- License & Copyright head           : Unified license header template
- bin/ obj/                          : Build artifacts (ignored)

Data flow summary:
- UI (Visuals) builds a simulation configuration via `SimulationRunner` and sends commands to `Backward/*.py` (e.g., `Hines_method.py`) through `PythonWorker` (Python.NET).
- `IOService` / `PythonWorker` handle Python environment discovery, GIL management, and asynchronous callbacks for long-running tasks.
- Simulation results are returned via JSON / numpy-serialized formats; `ExportUtils` and `DataManager` handle persistence and export.

Development notes:
- The biophysics and numerical core (`Backward/Hines_method.py`, `Visuals/Tabs/Modeling/Biophy.cs`) are informed by Destexhe (1998) and Hines (1984); these files include special scientific headers citing those works.
- External dependencies: `HelixToolkit.Wpf` (3D rendering), `pythonnet` (cross-language interop).
