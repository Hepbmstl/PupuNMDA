NeuronCAD (Root)
├── App.xaml / App.xaml.cs / AssemblyInfo.cs
│   └── WPF application entry, theme resources, Python shutdown on exit
│
├── NeuronCAD.csproj / NeuronCAD.slnx
│   └── .NET 8 WPF project configuration, HelixToolkit.Wpf + pythonnet references,
│       Backward/*.py copied to output directory
│
├── Backward                        // Namespace: NeuronCAD.Backward
│   ├── PythonWorker.cs
│   │   └── Dedicated Python worker thread, GIL management, Python DLL discovery
│   ├── SimulationRunner.cs
│   │   └── C# → Python simulation orchestration and plotting bridge
│   ├── SaveLoadManager.cs
│   │   └── ProjectData JSON serialization/deserialization and scene rebuild
│   ├── Hines_method.py
│   │   └── Multi-compartment solver, HH/CaT/GHK kinetics, probe export, phase portrait tools
│   ├── trans_geo_to_json tcD.py
│   │   └── NEURON .geo → NeuronCAD JSON converter for tcD-style morphology/biophysics
│   └── trans_geo_to_json tc200.py
│       └── NEURON .geo → NeuronCAD JSON converter for tc200-style region mapping
│
└── Visuals                         // Namespace: NeuronCAD.Visuals
    ├── Windows
    │   ├── MainWindow.xaml / MainWindow.xaml.cs
    │   │   └── Single shell for Modeling / Simulation / Reporting, menus, toolbars, progress UI
    │   ├── IonChannelSettingWindow.xaml / IonChannelSettingWindow.xaml.cs
    │   │   └── HH parameters, CaT parameters, global channel defaults
    │   └── SplashWindow.xaml / SplashWindow.xaml.cs
    │       └── Placeholder splash window
    │
    └── Tabs
        ├── Shared
        │   ├── SharedSceneState.cs
        │   │   └── Shared viewport, entity list, connections, devices, simulation registry, last simulation data
        │   └── IViewportInteractionHandler.cs
        │       └── Common viewport interaction contract
        │
        ├── Modeling
        │   ├── Viewport.cs
        │   │   └── Viewport environment, gestures, unprojection helpers
        │   ├── Interaction.cs
        │   │   └── Entity place / move / connect / select / delete interaction state machine
        │   ├── Panels.cs
        │   │   └── Entity property cards and per-entity ion-channel editing UI
        │   ├── Biophy.cs
        │   │   └── Global channel templates (Na, K, L, CaT)
        │   ├── ModelingPage.xaml / ModelingPage.xaml.cs
        │   │   └── Placeholder compatibility page, real functionality moved into MainWindow
        │   └── Visuals
        │       ├── IVisualEntity.cs
        │       │   └── Common entity and anchored-entity interfaces
        │       ├── VisualEntityBase.cs
        │       │   └── Common geometry/material/wireframe/channel-particle behavior
        │       ├── AxonVisual.cs
        │       │   └── Frustum/cylinder geometry, anchor conversion, DendVisual subclass
        │       ├── SomaVisual.cs
        │       │   └── Soma wrapper built on AxonVisual logic
        │       └── Connection.cs
        │           └── Connection data, visual lines, endpoint spheres, controller
        │
        ├── Simulation
        │   ├── SimulationRegistry.cs
        │   │   └── NSeg/LSeg compartmentalization, entity connection mapping, device binding
        │   ├── SimulationInteraction.cs
        │   │   └── Device placement, dragging, deletion, selection
        │   ├── SimulationPanels.cs
        │   │   └── Current clamp / probe / voltage clamp parameter panels
        │   └── attacheddevices.cs
        │       └── Attached device abstraction and concrete device types
        │
        └── Reporting
            ├── ReportingInteraction.cs
            │   └── Compartment overlays, entity selection, compartment picking
            └── ReportingPanels.cs
                └── Components / Probes report panels and plot triggers

Notes:
- The current repository does not contain IOService.cs, DataManager.cs, Entities.cs, ExportUtils.cs,
  Visuals/Shell/MainMenu, SimulationPage.xaml, or ReportPage.xaml.
- Modeling, Simulation, and Reporting all live inside MainWindow rather than separate shell pages.
- SplashWindow and ModelingPage are placeholders and should not be described as fully implemented modules.
- Tests are intentionally omitted from this structure summary.
